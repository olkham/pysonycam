Get ContentInfoList
Information about captured content can be checked via ContentInfoList. ContentInfoList is obtained using
SDIO_GetContentInfoList.
This operation retrieves information about captured content after a specified time and up to a specified
number. The number of contents per slot and the capture times can be checked using
SDIO_GetCapturedDateList.
When there are changes to the ContentInfoList, SDIE_ContentInfoListChanged is issued.
Captured content is assigned several IDs:
ContentID: A unique ID within a single slot. It is used when acquiring captured content.
UMID: A globally unique ID. It is used to identify already acquired data or to resume partially
transferred data.
Three types of time information related to ContentInfoList exist:
DataTime:
Exists within the ContentInfoList and represents the time the list itself was generated.
This time is not updated when content is added.
ContentInfoList regenerate time:
DeviceProperty is prepared for each slot.
Updated when the ContentInfoList stored in the camera is generated or regenerated.
This time is not updated when content is added.
ContentInfoList update time:
DeviceProperty is prepared for each slot.
Updated when the ContentInfoList stored in the camera is updated.
Updated in cases such as content addition, deletion, editing, or media insertion/removal.
When obtaining the ContentInfoList, you can determine whether the list is still valid by comparing
“DataTime” and “ContentInfoList regenerate time.”
Get Content
To retrieve content, use SDIO_GetContentData. For thumbnail/screennail data, use
SDIO_GetContentCompressedData.
Using the information obtained from Get ContentInfoList, specify the contentID, slotID, and fileID to
retrieve the desired captured content.
