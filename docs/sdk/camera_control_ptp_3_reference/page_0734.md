Firmware files can be transferred and executed in debug mode.
You can test the remote firmware update process without using firmware data by using debug mode. This
allow to test the feature before next firmware file is available.
```c
1. Set Debug Mode (0xE0D5) to value 0x02 “Pseudo-update with Fake Firmware”
```
2. Run the same firmware update sequence as in the normal sequence
```c
3. After the camera restarts, Debug Mode (0xE0D5) will automatically return to value 0x01 “OFF”
```
The camera update is complete immediately without an actual update.
Using a firmware file is not required to be a valid firmware file. You can use a dummy file.
```c
The return value of SDIO_GetFirmwareUpdateInfo (0x9247) will always be “9.99” and “OK”
SDIE_ControlUploadDataResult (0xC214) will always return “OK”
```
