Even if reconnection occurs during content file transfer, the process can resume from the point of
```c
interruption.
```
In the event of a disconnection during transfer, the “DataTime” of the obtained ContentInfoList should be
compared with the “ContentInfoList regenerate time.” If these values match, the transfer can be resumed.
However they differ, ContentInfoList must be reacquired, and the UMID of the already acquired data
should be compared with the new ContentInfoList to identify the appropriate file for resumption (any
partially transferred data must be discarded).
