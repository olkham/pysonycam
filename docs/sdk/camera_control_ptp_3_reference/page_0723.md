If content is deleted while retrieving the list,
valid content will be compacted toward the beginning of the list,
and the remaining areas will be filled with zeros.
Since content_type will be set to 0, this can be used to identify deleted content.
When there are changes to the content list, SDIE_ContentsInfoListChanged will be notified.
Please retrieve the list again as needed.
When retrieving the list during content saving,
the file_size of the content being saved may be 0.
In such cases, please retrieve the list again.
